Overview

This project showcases a two-step pipeline using a generic LLM agent for healthcare analytics. The first step generates answers and analytic outputs from free-text queries; the second step applies a separate LLM-based judge, guided by structured healthcare reporting and quantitative criteria, to score and explain the quality of those outputs.

Data & results are fully simulated. All statistics, figures, and examples in this project are synthetic and not representative of any real company, product, patient population, or commercial entity. They are provided solely to illustrate workflow and modeling capabilities.

Healthcare use cases

The same pattern can be used across a wide range of healthcare analytics problems:

  • Designing and reviewing clinical trial protocols and study reports.
  • Summarizing and stress-testing therapeutic outcome and real‑world evidence analyses.
  • Interpreting models for drug pricing, market access, and uptake scenarios.
Step 1

Query for Answer (LLM Agent)

A generic LLM agent ingests a research or analytics question, plans the steps, and orchestrates a multi-stage pipeline (query validation, data/feature extraction, statistical analysis, and visualization).

In the reference implementation, this is handled by a Python script that:

  • Receives a batch of healthcare analytics questions from JSON.
  • Uses an LLM to generate a clean case title / folder name for each query.
  • Runs a multi-step analysis pipeline (validation → data → stats → results).
  • Writes structured JSON reports that downstream evaluators can score.
llm_agent_pipeline.py ├─ unpack_question(...) ├─ run_analytics(question, user_id) │ ├─ QUERY_VALIDATION │ ├─ DATA_EXTRACTION │ ├─ STATS_ANALYSIS │ └─ VISUALIZATION → RESULTS └─ batch_run_analytics(...)
Step 2

LLM as a Judge (Healthcare Criteria)

A separate LLM-based judge evaluates each generated report against a structured healthcare reporting checklist and quantitative rules. The judge returns machine-readable scores and natural language justifications.

Two complementary evaluators are provided:

  • Healthcare reporting checklist judge: grades narrative findings against a detailed clinical-reporting checklist (study design, participants, outcomes, bias, interpretation, etc.).
  • Statistical test type judge: inspects model outputs (coefficients, p‑values, hazard ratios, etc.) and infers which statistical tests were used across many JSON results.
llm_judge_healthcare_criteria.py └─ grade_research_finding(...) → JSON scores per checklist item llm_judge_stats_type.py └─ get_statistical_tests_from_json(...) → inferred test types per file

Benchmark Report (Simulated)

The benchmark_report_generator.py script aggregates results across many questions and batches and produces a Word report with simulated statistics and healthcare-focused summaries.

  • End‑to‑end success rates and runtime distributions across batches.
  • Coverage and consistency relative to a generic healthcare reporting criteria checklist.
  • Failure patterns and diagnostic views for problematic questions.

All key metrics in the overview and figures are randomly perturbed within realistic ranges to ensure they do not reveal any operational performance data.
